# Local Mode First-Use Activation

OB-GOAL-026 把下一阶段目标收敛为一件事：普通用户安装桌面版后，能在几分钟内知道怎么开始，并看到第一份有依据的今日整理预览。

## 主路径

首次打开桌面版时，未完成激活前不进入主应用。用户只能先选择：

- 先看样例：进入样例体验，明确未读取真实数据。
- 启用本地模式：进入完整激活流程。
- 稍后回来：停留在引导页，不进入空控制台。

本地模式固定为六步：

1. 了解 OpenButler 能帮什么。
2. 确认隐私承诺：只读、默认本地、不复制截图、不外发。
3. 配置智能整理钥匙。
4. 扫描或安装本机记录组件。
5. 授权前预览将读取什么。
6. 用户确认后生成今日整理预览。

## 用户语言

普通界面使用“智能整理钥匙”和“本机记录组件”。高级说明里可以说明底层项目名是 MineContext。

普通界面不显示：

- MineContext
- PCActivity
- mock
- seed
- Provider
- Webhook
- 本地截图路径
- 原始事件 id

## 授权前预览

授权前只允许显示聚合信息：

- 时间范围
- 预计记录数量
- 数据类型
- 写入状态
- 隐私边界

确认前不写数据库，不导入真实事件，不显示活动标题、窗口标题、URL、截图路径或 raw output。

## 第一份今日整理预览

连接成功后，OpenButler 应先给用户一张“今日整理预览”主卡。它回答：

- 今天整理了什么。
- 哪条最值得先看。
- 依据来自哪里。
- 哪些结论还不能确认。

如果数据不足，界面要说明下一步怎么补齐，而不是显示空控制台。

## 当前边界

本轮不读取真实 MineContext 活动，不复制截图，不调用外部模型，不新增云端能力。

完整安装包、托盘唤起、卸载时后台进程清理仍需要单独做桌面 packaged 回归。
