# Today as Command Center

OB-GOAL-012 把 `/butler` 首页收敛为“今日中控”。首页不再承担展示所有模块的责任，只回答三个问题：

- 今天有什么需要先看？
- 现在处于什么数据模式？
- 下一步点哪里？

## 信息层级

移动端第一屏按这个顺序展示：

1. 轻量模式提示，例如“样例体验，未读取你的真实数据”或“只在本机整理”。
2. 今日摘要主卡，用一句话说明今天的状态。
3. 一个主行动，例如“看今天建议”或“先看样例”。
4. 2 到 3 个关键数字。
5. 最高优先级建议。

完整指标、场景信号和高级能力不放在第一屏。它们下沉到“更多今日信息”和“高级与实验室”。

## 文案规则

首页文案按 `humanizer-zh` 的标准处理：

- 少用口号，不说“核心价值”“全场景赋能”。
- 不解释内部实现，先说用户能做什么。
- 不把样例写成真实记录。
- 不在普通页面出现 `MineContext`、`PCActivity`、`UnifiedTimelineEvent`、`mock`、`seed`、`Provider`、`Webhook`。

推荐文案：

- 标题：今天先看这几件事
- 状态：有 3 条样例记录值得回看，1 条建议适合先处理。
- 样例提示：样例体验，未读取你的真实数据。
- 本地提示：只在本机整理。

## 空状态

无数据时，首页不显示空控制台。它只给用户三个选择：

- 先看样例
- 连接本地数据源
- 稍后配置

OpenButler 不用空数据编造今日结论。

## 验收标准

- 390x844 移动视口下，第一屏能看到标题、今日状态、主行动和首要建议。
- 页面无横向滚动。
- 最高优先级建议在首屏或首屏下沿可见。
- 隐私提示存在，但不抢主视觉。
- 高级能力保持折叠。
- 本轮不新增后端 API，不读取真实 MineContext 数据，不调用外部模型。
