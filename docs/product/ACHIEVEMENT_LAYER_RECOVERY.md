# Achievement Layer Recovery

OpenButler 的成就系统是一个正反馈层，不是绩效系统。

它的作用是把用户一天里值得留下的小进展整理出来：一段稳定专注、一次待办收尾、一次按时休息、一次复盘、一次可自动化流程发现。页面只帮助用户看见进展，不评价人格，不制造焦虑，也不把娱乐或休息写成负面。

## 产品入口

- 主导航新增 `成就`，路径为 `/achievements`。
- `今日` 首页保留 `今天的小成就` 轻量入口。
- `时间线` 继续保留 `成就记录` 筛选。
- `问管家` 继续支持“本周有哪些小成就”一类问题。

## 页面结构

成就页 V1 分成三块：

1. `今天的小成就`：展示已经整理出的成就卡。
2. `连续记录`：展示连续复盘、稳定投入等温和趋势。
3. `下一枚可解锁`：给出一个低压力下一步，例如晚间回看或了解本地模式。

每条成就卡默认显示标题、说明、时间、来源和可信度。依据默认折叠，用户点击后才看到来源、边界说明和隐私说明。

## 数据边界

- 不新增后端 API。
- 不读取真实 MineContext 数据。
- 不复制截图。
- 不上传数据。
- 不调用外部模型。
- 公开站点可以使用样例成就，但必须明确：`样例体验，未读取你的真实数据。`

真实本地模式下，成就只能来自用户已授权的本地线索整理。

## 禁止事项

普通用户界面不显示：

- `achievement`
- `event_type`
- `PCActivity`
- `source_event_id`
- `raw_ref`
- `evidence_refs`
- `mock`
- `seed`
- 本地截图路径

高级与实验室可以保留调试信息，但不能进入成就页普通路径。

## 后续路线

V1 只恢复展示与回看。后续可以考虑：

- 成就偏好设置
- 周/月成就回顾
- 用户手动确认成就
- 与目标页联动

暂不做积分、等级、徽章商城、社交分享。
