# Butler Inbox as Decision Queue

OB-GOAL-014 把 `/butler/inbox` 改成决策队列。用户来这里不是看一堆通知，而是处理几件需要确认的事。

## 页面应该先回答什么

- 现在有什么需要我处理？
- 哪些提醒已经放到稍后？
- 哪些已经处理过？
- 哪些被我标成不准确？

默认进入“待确认”。其他状态不抢视线。

## 用户可见状态

| 后端状态 | 用户看到 |
| --- | --- |
| `new` / `seen` | 待确认 |
| `snoozed` | 稍后 |
| `accepted` / `resolved` / `dismissed` | 已处理 |
| `marked_inaccurate` | 不准确 |

页面不显示这些后端状态名。普通用户只看到中文状态。

## 用户动作

| 按钮 | 后端反馈 |
| --- | --- |
| 处理了 | `accepted_action` |
| 稍后再看 | `snoozeInsight(id, 60)` |
| 不准确 | `inaccurate` |
| 少提醒类似内容 | `too_frequent` |
| 有用 | `useful` |

操作成功后，卡片进入对应状态。保存失败时，卡片留在原处，并显示“没有保存成功，再试一次”。

## 降噪

如果用户标记“少提醒类似内容”，页面显示：

> 已记录。类似提醒后面会少出现。

后端已有规则会降低同类提醒优先级。前端只展示这个结果，不额外发明新的规则。

## 保护类提醒

`data_quality_notice` 和 `privacy_notice` 不能被永久静默。它们可以降低频率，但仍会在必要时出现。

用户看到的文案是：

> 隐私和数据质量提醒不会被永久关闭。

## 依据

默认不展开依据。用户点击“查看依据”后，只显示：

- 来源
- 类型
- 状态
- 可信度
- 边界说明
- 隐私说明

普通页面不显示 `insight_id`、`feedback_type`、`evidence_refs`、`raw_ref`、`source_event_id`，也不显示本地截图路径。

## 本轮边界

本轮不新增后端 API，不读取真实 MineContext，不复制截图，不调用外部模型。
